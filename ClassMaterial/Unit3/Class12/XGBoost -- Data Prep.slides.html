Model Prep: Handling Missing Values¶

This is a very common issue in working with data. Luckily tree-based models are the most resilient in handling them, due to their non-assumptive nature.

Key questions to ask yourself when handling missing values:

  • Is their occurrence random or not?
    • If they are random: You want to impute (ie, estimate) them
    • If they are not random: You want to encode them with their implied value

Non-Random Missing Values¶

This means the missing value has a specific meaning:

  • It never happened
  • 0
  • This thing doesn't exist

  • Absence, etc

    In this case, it usually is best to fill them in with a single value that represents what that value means.

Denoting A Missing Value¶

Often if you have a dataset with a lot of missing values, it is helpful to note that a particular value was originally missing. It helps a model "see" whether or not something was there.

In tree based models, this allows them to make more meaningful splits higher up, allowing it to converge with a fewer number of boosting rounds.

In [111]:
# a useful function to help you with this
def denote_null_values(df):
    empty_cols_query = df.isnull().sum() > 0
    empty_df_cols = df.loc[:, empty_cols_query].columns.tolist()
    for col in empty_df_cols:
        col_name = f"{col}_missing"
        df[col_name] = pd.isnull(df[col])
    return df

What if you have randomly missing values?¶

This could happen if your data was collected online, and your server suddenly went down. There was an unusual weather incident that caused a business process to shut down, someone at your job quit or got fired, etc.

Here you must impute.

  • Simple rule: use the average value for numeric columns, the modal (most frequent) value for categorical data
  • More complicated rule: use a model to predict them!

EncodingText Based Data¶

Three primary ways of doing this:

  • onehot / dummy encoding: create a binary representation of your different category values
  • ordinal encoding: assign numeric values according to something's rank
    • ie, Small, Medium, Large --> 1, 2, 3
  • category encoding -- just randomly assign numbers to each unique value in a dataset

Which One Is Better?¶

For most models, onehot encoding or ordinal encoding is best, because it gives you a more granular representation of your data.

However, for tree based models, it tends not to make a difference, and there is probably a slight preference for regular encoding.

Why?

  • Categorical variables with lots of values create sparse columns, that result in misshapen trees
  • It obscures features importance after the model is fit
